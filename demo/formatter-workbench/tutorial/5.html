Data editor

This editor let’s you adjust the data shown in the grid.
  • The data is stored as an array of dataRow objects.
  • The initial contents reflects the actual data shown in the grid instance above it.
  • After editing the data, bind it to the grid by clicking grid.setData(…).
  • The grid in turn is bound to the data window: Any edits made to cells (via cell editors) will alter the contents of this editor.

In actual practice, when data is set using setData, it typically has come from a data store, typically as a character string in JSON format. If that is the case, you would probably call JSON.parse on it, rather than eval; and pass the result to setData.

In this workbench, we’re using straight JavaScript, not JSON, which is why we’re using using eval, not JSON.parse. When editing, therefore, you do not have to worry about strict adherence to the JSON standard (double quotes, primitive types, _etc._).

Activity 1: Edit a grid cell; the data tab changes

Add a comma between "Apple" and "Inc." in the data tab:
  1. Double click "Apple, Inc." in the grid.
  2. Remove the comma.
  3. Press the enter key.
  4. Observe how both the grid cell and the data tab update.

Activity 2: Edit a value in the data tab; the grid updates

Remove the comma from the grid cell:
  1. Click the Data tab.
  2. Edit the name property in the first (dataRow) object.
  3. Click the grid.setData(…) button.
  4. Observe how the grid cell updates.

Activity 3: Add a row in the data tab; the grid updates

Add a comma between "Apple" and "Inc.":
  1. Click the Data tab.
  2. Copy and paste the last (dataRow) object so now there are two of them.
  3. Optional: Edit the values in the last one to make it distinct.
  4. Click the grid.setData(…) button.
  5. Observe how the new row is added to the grid, which now has a total of five rows.
